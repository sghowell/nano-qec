# Docs Review Checklist

This checklist is the final pass for human-facing documentation changes. It is
intended to keep the docs precise, navigable, and aligned with the repo’s
authority model before a change is merged.

## Implemented Today

As of April 9, 2026, this checklist applies to `README.md` and the human-facing
docs under `docs/human/`.

## Future Direction

If the docs surface grows materially, expand this checklist rather than
silently relying on tribal memory.

## Checklist

- Does the page have one obvious audience?
- Does the opening paragraph say what the page is for?
- Does the page distinguish implemented behavior from future direction?
- Are dates explicit where claims depend on timing?
- Are benchmark names explicit where results are discussed?
- Are commands copy-pasteable and actually validated?
- Do links resolve?
- Do images render on GitHub?
- Does the page avoid redefining protected CLI, schema, or mutation-policy
  rules?
- If the page summarizes authority docs, does it link back to them?
- Does the page end with clear “Next reads” links?
